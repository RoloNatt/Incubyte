For this prototype, optimize for simplicity, fast development, and low setup overhead — not scalability or enterprise architecture.

1. Backend: FastAPI
Why FastAPI?
Pros
Very simple to build APIs

CRUD endpoints are quick to write:

Create employee
Get employees
Update employee
Delete employee
Salary insights endpoints

Less boilerplate than many alternatives.

Built-in validation

If salary is negative or a required field is missing, FastAPI handles validation automatically.

This saves code.

Works well with React

FastAPI naturally exposes JSON APIs, which React consumes easily.

Easy local development

Just run:

uvicorn main:app --reload

No complicated setup.

Automatic API docs

FastAPI gives Swagger UI automatically.

Useful for testing backend quickly during development.

Why not Flask?

Flask is also fine, but:

More manual validation
More boilerplate
More setup for structured APIs

FastAPI feels cleaner for a CRUD prototype.
