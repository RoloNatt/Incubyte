This document contains the key prompts used to develop the Salary Management Tool, formatted for easy reference.

## Prompt 1: Initial Requirements

**Date:** 2026-05-23

We need to build a salary management tool for an organization of 10,000 employees. The user persona is the HR manager of the organization. We need to build both frontend and backend but the app will currently be on localhost. The app must be simple, clean, and usable, and end-to-end fully functional. Do not write any code now, we need to discuss about the app first.

### Requirements

**Managing Employees**
- Add, View, update, and delete employees via UI
- The employee must have a full name, job title, country, salary, along with any other meaningful data that you believe should be captured

**Salary Insights via UI**
- Minimum, maximum, average salary of employees in a country
- Average salary for the given Job Title in a country
- Any other meaningful metrics you believe are helpful for the user persona

### Technical Constraints

You should build an end-to-end, fully functional software, including backend & UI.

**Backend**
- Python
- Database - SQLite

**UI**
- ReactJS or NextJS
- Component library of your choice

**Seeding**
- Seed script with 10,000 employees
- Full names should be generated by combining values from first_names.txt and last_names.txt
- Assume that engineers run this script regularly, and performance of the script matters

## Prompt 7: Final Product-Level Improvements

The Salary Management Tool is working and visually much better now, but it needs **final product-level improvements** to make it feel like a polished HR compensation and budgeting dashboard.

### Key Requirements

**1. Global Salary Period Toggle (Annual/Monthly)**
- Add a toggle on both Employees and Insights pages
- When switched to Monthly, all salaries should display as monthly values (annual / 12)
- When switched to Annual, all salaries should display as annual values
- This must affect:
  - Employee table salary display
  - All compensation cards
  - All charts
  - All analytics
- Add a visual indicator showing the current period (e.g., "/mo" or "/yr" labels)

**2. Enhanced Salary Display**
- Make salary values bold and prominent
- Add period labels to salary displays
- Add tooltips showing both annual and monthly values on hover
- Improve formatting with proper currency symbols and number formatting

**3. New Compensation Metrics**
Add these metrics to the insights dashboard:
- Min salary
- Median salary (new)
- Average salary
- Max salary
- Employee count
- Monthly payroll (new)
- Annual payroll (new)
- Payroll share % (new - what % of global payroll this country represents)
- VS Global Avg with clearer display (▲/▼ with percentage and "above/below global average" text)

**4. Insight Summary Strip**
Add an auto-generated summary strip at the top of the Insights page showing:
- Payroll share % for selected country
- Top paying department
- Largest department (by headcount)
- Top paying job title
These should be auto-generated from the data and provide instant context.

**5. Fix Department Chart Y-Axis Bug**
The department comparison chart Y-axis is not formatting numbers properly. Fix this to use proper number formatting (K, M, L, Cr).

**6. Chart Visual Improvements**
- Use a softer, more professional color palette
- Reduce chart heights for better layout
- Add dynamic subtitles to charts that show contextual insights (e.g., "Most employees fall in X range", "Engineering consumes highest budget")
- Fix any axis formatting issues

**7. Replace Pie Chart with Horizontal Bar Chart**
The employee distribution pie chart should be replaced with a horizontal bar chart for better readability and comparison.

**8. Add Payroll by Department Chart**
Add a new chart showing total payroll by department. This is critical for budgeting decisions.

**9. Improve Job Title Salary Section**
Enhance the job title salary section to show:
- Median salary
- Headcount
- Monthly payroll cost
- Annual payroll cost
- VS country average comparison

**10. Improve Stats Card Hierarchy**
- Add icons to stats cards for visual recognition
- Use color to emphasize important cards (e.g., average salary in primary color)
- Add secondary text for additional context
- Create visual hierarchy between compensation cards and budget cards

**11. Reorganize Insights Page Layout**
Organize the Insights page as follows:
- Filters at top (country, job title, currency, period toggle)
- Summary insight strip (auto-generated)
- Compensation cards section (min, median, avg, max)
- Budget cards section (employees, monthly payroll, annual payroll, vs global avg)
- Job title insight section (when job title selected)
- Key charts section (salary distribution, payroll by department)
- Collapsible "Detailed Analytics" section containing:
  - Job title comparison chart
  - Employee distribution chart
  - Department comparison chart

**12. Table Improvements**
- Add country flags (🇮🇳, 🇺🇸, etc.) next to country names
- Add salary tooltip showing both annual and monthly values
- Add period label (/yr or /mo) to salary display
- Improve row hover effect for better interactivity
- Make sort indicators more visible

**13. Backend API Enhancements**
Update backend APIs to return the new metrics:
- Country insights: median, annual_payroll, monthly_payroll, payroll_share, top_department, top_job_title, largest_department
- Job title insights: median, count, annual_payroll, monthly_payroll, vs_country_avg
- Add department payroll API endpoint

**14. Number Formatting Everywhere**
Ensure consistent number formatting across:
- Cards
- Charts
- Tooltips
- Table
- Labels
- Axes
Use Indian number format (lakhs, crores) for INR and Western format (K, M) for other currencies.

### Important

Do NOT overdesign.

Make it feel like a **professional HR compensation dashboard** where the HR manager does not need to use a calculator because all meaningful salary and payroll insights are already visible on screen.
